# Microsoft 365 Administration Lab

## Overview

This project demonstrates hands-on administration of a Microsoft 365 Business tenant. The lab focuses on common Microsoft 365 and Microsoft Entra ID administration tasks that are frequently performed by IT Support and Helpdesk professionals.

The purpose of this project is to gain practical experience managing cloud identities, licenses, groups, mailboxes, authentication, and administrative roles in a Microsoft 365 environment.

## Key Concepts Learned

### Shared Mailbox vs Distribution List

#### Shared Mailbox

Example:

[itsupport@company.com](mailto:itsupport@company.com)

Used when multiple users need access to the same mailbox and can read, manage, and respond to emails together.

#### Distribution List

Example:

[all-it@company.com](mailto:all-it@company.com)

Used to distribute emails to multiple recipients. Each member receives their own copy of the email in their mailbox.

### On-Premises vs Cloud Identity

#### Active Directory

* Hosted on Windows Server
* Managed on-premises
* Example domain: `lab.local`

#### Microsoft Entra ID

* Hosted in Microsoft Cloud
* Managed through web portals
* Example domain: `phyosithuhein.onmicrosoft.com`
